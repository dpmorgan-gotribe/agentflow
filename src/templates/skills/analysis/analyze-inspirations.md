# Analyze Inspirations

Create a mood board and collect reference designs.

## Output Requirements

OUTPUT ONLY RAW MARKDOWN. No explanations. No descriptions.

Your response must:
- Start with: `# Design Inspirations`
- Be valid Markdown content
- Follow the output format below exactly

DO NOT:
- Explain what you're creating
- Ask for permission
- Wrap in markdown code fences (```)
- Add any text before or after the markdown
- Say "Now I have..." or "Let me..." or "Here's the..."

## Inputs
- Brief (tone, brand personality)
- Competitive research (competitor visuals)
- Style direction hints

## Process

1. **Define Mood**:
   - What feeling should the app evoke?
   - What keywords describe the desired experience?
   - What should be avoided?

2. **Find References**:
   - Search for designs that capture this mood
   - Dribbble shots
   - Behance projects
   - Real app screenshots
   - Design system examples

3. **Extract Patterns**:
   - What visual patterns appear across inspirations?
   - What interactions are common?
   - What makes these designs successful?
