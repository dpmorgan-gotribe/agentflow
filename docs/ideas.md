- [ ] 2026-05-08 ‚Äî `.gitattributes` to normalize CRLF/LF on Windows worktrees. Empirical: per-bug worktrees on Windows show every file as "modified" via `git status --short` due to `core.autocrlf=true` line-ending normalization. Diff content (`-w` / whitespace-ignoring) is empty, but: (a) noise in operator inspection, (b) larger orchestrator stdout when git diff stat returns 100+ files, (c) potential false-positive trip on any future "diff size" heuristic. Fix: add `* text=auto eol=lf` to `.gitattributes` factory-template + each shipped project. ~5 min change. Validation: post-fix, `git status --short` in a fresh per-bug worktree returns only files the orchestrator's seedWorktree intentionally edits (`.claude/settings.json`).
